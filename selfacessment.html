2. Design and implement 4×4 memory using single cell memory

Introduction

A memory cell is the fundamental unit of RAM. A single static RAM (SRAM) cell stores one bit of data. By arranging such cells in rows (word lines) and columns (bit lines), we build a complete memory array. This design uses single SRAM cells to build a 4×4 memory — 4 locations each storing 4 bits.

Working of a Single SRAM Cell

Components:

  • Two cross-coupled CMOS inverters (INV1 and INV2), forming a bistable latch
  • Two NMOS access transistors (N1 and N2), controlled by the Word Line (WL)

Ports:

  • WL (Word Line): Activates the cell for read/write
  • BL (Bit Line): Carries data during write; read ones during read
  • BL̄ (Complement Bit Line): Carries the inverted data bit

Reading/Writing Process:

The cross-coupled inverters make nodes Q and Q̄ complementary. During write, Q and Q̄ are driven to BL and BL̄ respectively. When WL is HIGH, connecting the inverter nodes to BL and BL̄. During write, the sense amplifier drives BL/BL̄ forcefully to override the stored data. The stored charge directly drives BL/BL̄ slightly, detected by the sense amplifier.

4×4 Array Operation:

The 4×4 array (shown in diagram) has 4 rows and 4 columns:

  • A 2-to-4 row decoder selects a 2-bit address (A[1:0]), and drives one WL (WL[0-WL3]) at a time
  • All 4 bit lines (BL0-BL3) carry one bit each, forming a 4-bit data word
  • Write: Address selects WL, cells discharge their BL lines; sense amplifiers read Z_out[3:0]
  • Read: Address selects WL, data is driven to BL/BL̄ lines

Simulation Response Design:

Write 0xAB (10101011) to address 01 → WL1 goes HIGH → Bank 1: C1D = gives HIGH, C1[0] = 1, C1[1] = 0. Reading address 01 → Data from = 0x0B. Read back → Data from 1 = 0xAB SHEET.

Conclusion:

The 4×4 SRAM uses 16 single-bit SRAM cells arranged in a matrix. The row decoder selects the word line, and the bit-line sense amplifiers complete the read/write operation. This technique is extensively used in practical memory subsystem design.

3. Design and implement a larger memory of size 1024×32 memory using a smaller memory of size 512×16.

Introduction

Large memory systems are built by combining smaller memory chips. A 512×16 chip has 512 addressable locations each storing 16 bits. We need to build a 1024×32 memory — 1024 locations each 32 bits wide (32 bits). This requires expanding both word depth (512→1024) and word width (16→32 bits).

Design Steps

Step 1 — Determine expansion needed:

  • Target: 1024 × 32 available modules: 512 × 16
  • Width expansion: 32 ÷ 16 = 2 chips in parallel per bank to get 32 data bits
  • Depth expansion: 1024 ÷ 512 = 2 banks to get 1024 addresses
  • Total chips needed: 2 × 2 = 4 chips

Step 2 — Address partitioning:

  • Total address bits = log₂(1024) = 10 bits: A[9:0]
  • A[8:0] (9 bits) go to all chips as row address (512 locations per chip)
  • A[9] (MSB) is fed to a 1-to-2 decoder to generate Chip Select (CS0, CS1):
    • A[9] = 0 → CS0 = 1 → Bank 0 selected (addresses 0 – 511)
    • A[9] = 1 → CS1 = 1 → Bank 1 selected (addresses 512 – 1023)

Step 3 — Data bus connection:

  • Chip A and Chip C provide D[15:0] (lower 16 bits)
  • Chip B and Chip D provide D[31:16] (higher 16 bits)
  • Both chips in the same bank are enabled simultaneously by the same CS signal

Step 4 — R/W signal:

Connected to all chips simultaneously. A single READ or WRITE operation accesses both chips in the selected bank, reading/writing all 32 bits at once.

Read and Write Operations:

Write: CPU places A[9:0] on address bus, data D[31:0] drives all 32-bit data lines. R/W = 0 (write). Decoder selects CS for correct bank. R/W = 0. Selected chip drives D[15:0] and D[31:16] into the addressed locations simultaneously.

Read: CPU places A[9:0] on address bus. Decoder asserts CS for correct bank. R/W = 1. Selected chip drives D[31:0] and D[31:16] onto the bus. CPU reads combined 32-bit word.

Simulation Response Design:

Write 0x4BEEF to address 0x200 (512 in decimal) → Bank 1, CS1 gives HIGH, A[8:0] = 0. Chip C stores 0x4E:AB, Chip D stores 0x4E:EF. Read back → Data from = 0x4E:BEEF.

Conclusion:

A 1024×32 memory is constructed from four 512×16 chips by width-expanding (chips in parallel) and depth-expanding (banks with chip-select decoding). The MSB of the address selects the bank; the remaining bits address the row within the bank. This technique is extensively used in practical memory subsystem design.

4. Design and implement both alternatives of a dual-ported 8-bit register file having 8 registers.

Introduction

A register file is a fast internal memory inside a processor, holding a small number of registers used by the CPU during instruction execution. A dual-ported register file supports two simultaneous read operations (one read and one write), which is essential for ALU instructions that need two source operands at the same time (e.g., Add R1, R2, R3).

Design Specification

  • Number of registers: 8 (R0–R7)
  • Width of each register: 8 bits
  • Ports: 2 read ports (RA, RB) + 1 write port (RW)
  • Address bit width: log₂(8) = 3 bits per port

Address / Port Width Table

Width Description
3 bitsSelect register for Read Port A
3 bitsSelect register for Read Port B
3 bitsSelect register for Write Port
8 bitsData to be written
1 bitWrite Enable
8 bitsOutput from Read Port A
8 bitsOutput from Read Port B

5. Design and implement the hardware needed to compute the following instructions:
(i) Add R0, R1, R2     (ii) Add R0, R1, #0x45

Introduction

In a processor, instructions are executed using a datapath, which connects the register file, ALU, and memory units. This question requires designing hardware to execute the following instructions:

  1. Add R0, R1, R2 → R0 = R1 + R2
  2. Add R0, R1, #0x45 → R0 = R1 + Immediate (0x45)

The design must show how data flows between components and how instructions are executed step-by-step.

Design Overview

The required hardware consists of:

1. Register File

  • 8 registers (R0–R7), each 8-bit
  • 2 read ports and 1 write port

2. ALU

  • Performs addition operation

3. Multiplexer (MUX)

  • Selects between register data and Immediate value

4. Code Memory

  • Stores Instructions

5. Control Unit

  • Generates control signals

Block Diagram Explanation

Draw the diagram with the following labeled components:

Main Blocks:

1. Code Memory

  • Outputs: Instruction

2. Register File

  • Inputs:
    • Read address A (R1)
    • Read address B (R2)
    • Write address (R0)
    • Write Data
    • Write Enable (WE)
  • Outputs:
    • Data A
    • Data B

3. Multiplexer (MUX)

  • Input 1: Data B (from register)
  • Input 2: Immediate value (Imm1)
  • Control: Select signal

4. ALU

  • Inputs:
    • Operand A (Data A)
    • Operand B (MUX output)
  • Outputs:
    • Result

5. Control Unit

  • Controls:
    • MUX select
    • Register write enable
    • ALU operation

Tracing of Instructions

1. Instruction: Add R0, R1, R2

Step-by-Step Execution:

  1. Instruction is fetched from code memory
  2. Register file reads:
    • R1 → Data A
    • R2 → Data B
  3. MUX selects Data B (register value)
  4. ALU performs:
    • Result = R1 + R2
  5. Result is written back to R0

2. Instruction: Add R0, R1, #0x45

Step-by-Step Execution:

  1. Instruction is fetched from code memory
  2. Register file reads:
    • R1 → Data A
  3. Immediate value (0x45) is extracted
  4. MUX selects Immediate value
  5. ALU performs:
    • Result = R1 + Imm1
  6. Result is written back to R0

Control Signals Table

Instruction MUX Select ALU Operation Write Enable
Add R0, R1, R2 0 (Register) ADD 1
Add R0, R1, #0x45 1 (Immediate) ADD 1
